# How to Export and Transfer the Codebase to a New Windows Machine

---

## What to Include and What to Exclude

Never copy `node_modules` or `.expo` — they are machine-specific and huge.

| Folder / File | Include? | Why |
|---|---|---|
| `SmartDeskAssistant/src/` | YES | All app source code |
| `SmartDeskAssistant/assets/` | YES | Images, icons |
| `SmartDeskAssistant/app.json` | YES | Expo config |
| `SmartDeskAssistant/App.tsx` | YES | App entry point |
| `SmartDeskAssistant/index.ts` | YES | App entry point |
| `SmartDeskAssistant/package.json` | YES | Dependency list |
| `SmartDeskAssistant/package-lock.json` | YES | Locked versions |
| `SmartDeskAssistant/tsconfig.json` | YES | TypeScript config |
| `SmartDeskAssistant/eas.json` | YES | EAS build config |
| `SmartDeskAssistant/node_modules/` | **NO** | 500MB+, regenerated by `npm install` |
| `SmartDeskAssistant/.expo/` | **NO** | Machine-specific Metro cache |
| `backend/` | YES (entire folder) | Docker builds everything inside |
| `DEVICE Code/` | YES | Arduino ESP32 firmware |
| `docs/` | YES | This guide and others |

## Step 1 — Create the Clean Zip (Run on the SOURCE Machine)

Open PowerShell and paste this **entire command as one line**:

```powershell
Compress-Archive -Path "C:\IIT\SDA-main\backend","C:\IIT\SDA-main\SmartDeskAssistant\app.json","C:\IIT\SDA-main\SmartDeskAssistant\App.tsx","C:\IIT\SDA-main\SmartDeskAssistant\index.ts","C:\IIT\SDA-main\SmartDeskAssistant\demo.js","C:\IIT\SDA-main\SmartDeskAssistant\package.json","C:\IIT\SDA-main\SmartDeskAssistant\package-lock.json","C:\IIT\SDA-main\SmartDeskAssistant\tsconfig.json","C:\IIT\SDA-main\SmartDeskAssistant\eas.json","C:\IIT\SDA-main\SmartDeskAssistant\IMPLEMENTATION.md","C:\IIT\SDA-main\SmartDeskAssistant\README.md","C:\IIT\SDA-main\SmartDeskAssistant\restoration.sql","C:\IIT\SDA-main\SmartDeskAssistant\src","C:\IIT\SDA-main\SmartDeskAssistant\assets","C:\IIT\SDA-main\DEVICE Code","C:\IIT\SDA-main\docs" -DestinationPath "C:\IIT\SDA-main-clean.zip" -Force
```

Verify the size — must be under 50 MB:

```powershell
(Get-Item "C:\IIT\SDA-main-clean.zip").length / 1MB
```

Expected output: around `8` to `15` MB. If it shows over 200 MB, `node_modules` was accidentally included.

The zip is saved at: `C:\IIT\SDA-main-clean.zip`

---

## Step 2 — Transfer the Zip

Copy `C:\IIT\SDA-main-clean.zip` to the new machine using any method:

- USB drive
- Google Drive / OneDrive / any cloud storage
- LAN file share
- Email (if under 25 MB)

---

## Step 3 — Extract on the New Machine

Open PowerShell on the new machine:

```powershell
# Create the destination folder first
mkdir C:\IIT

# Extract the zip
Expand-Archive -Path "C:\IIT\SDA-main-clean.zip" -DestinationPath "C:\IIT\SDA-main"
```

Verify the folder structure looks like this:

```
C:\IIT\SDA-main\
  backend\
  SmartDeskAssistant\
  DEVICE Code\
  docs\
```

---

## Step 4 — Install Frontend Dependencies (New Machine Only)

The `node_modules` folder was excluded from the zip. Recreate it:

```powershell
cd C:\IIT\SDA-main\SmartDeskAssistant
npm install
```

Takes 2-3 minutes. You will see packages being downloaded. This is normal.

You do NOT need to run `npm install` for the backend — Docker handles that automatically.

---

## Step 5 — Follow the Full Setup Guide

Now follow `NEW_MACHINE_SETUP.md` from PART 1 onwards:

1. Install prerequisites (Node.js, Docker Desktop, Expo Go)
2. Find your WiFi IP with `ipconfig`
3. Add Windows Firewall rules (run as Admin)
4. Start Docker: `docker compose up -d`
5. Login to Expo: `npx expo login` (username: `testvicha`)
6. Start Metro: `$env:REACT_NATIVE_PACKAGER_HOSTNAME="YOUR_IP"` then `npx expo start --go --lan --port 8083 --clear`
7. Scan QR with Expo Go on your phone
8. Register in app, add device (`test123`), connect Protonest

## Common Mistake: Running Commands Before `cd`

PowerShell opens in `C:\Windows\System32` by default. Always `cd` to the correct folder before running any project commands:

```powershell
# For backend:
cd C:\IIT\SDA-main\backend

# For frontend / Expo:
cd C:\IIT\SDA-main\SmartDeskAssistant
```

If you see `The expected package.json path: C:\Windows\System32\package.json does not exist` — this is why.
